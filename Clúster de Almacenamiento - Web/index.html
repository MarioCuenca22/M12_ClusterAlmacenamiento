Introducción

Esta página web está diseñada específicamente para poder facilitar la accesibilidad de los recursos que yacen en este nuestro proyecto. Aquí podrás encontrar los siguientes apartados:.

  • Arch Linux

    • Tanto esta página web como este servidor están alojados en el sistema operativo llamado Arch Linux, un sistema base que desde Kernel partimos hacia un diseño completamente personalizado y adaptado a nuestras necesidades, de esta manera podemos aprovechar al máximo el rendimiento de nuestro servidor gracias a instalar los paquetes únicamente necesarios para el funcionamiento de este.

  • Raspberry Pi Zero / 4

    • En el hardware de este proyecto podremos observar que disponemos de diversas Raspberry Pi de diversos modelos, tanto como una PI Zero como una PI 4 Modelo B que dispone de 8GB de memoria RAM, esta, la que será la Raspberry general que controla diversos módulos. En estas Raspberrys podemos apreciar que cada una dispone de una función diferente que se especifica de una manera mucho más concreta en la documentación.

  • Dominios y Apache

    • Como se podrá observar a los contenidos de nuestro proyecto se puede acceder mediante el dominio espetoneta.es tanto esta web como otros componentes, (véase webmin) lo podemos encontrar alojado en Apache3, un servicio gratuito ofrecido por la empresa Apache Software Foundation mientras que el contenido está mantenido en el dominio alquilado por la empresa OVH con la extensión ".es".

  • BOT Telegram

    • Nuestro proyecto consta como se ha mencionado anteriormente con diversas Raspberry Pi, en una de ellas alojaremos un BOT de Telegram, con este bot podremos tanto recibir como enviar datos / comandos / paquetes o incluso controlar directamente el servidor, usando el gestor de bots BotFather y el lenguaje de programación de Python 3.0.X. La idea principal de poder hacer uso de este bot es de poder mantenernos informados desde cualquier parte con cualquier dispositivo que tenga acceso a internet y Telegram instalado.

  • Grafana

    • Grafana es un gestor de Dashborads o más concido como paneles en los que podemos poner todo tipo de información, desde gráficas del costo eléctrico hasta usos de CPU, RAM entre otros datos altamente interesantes y de utilidad que podremos recurrir para comprobar que todo en nuestro sistema funciona correctamente.

Nuestro servidor

Queremos informarles que actualmente no contamos con un servidor para alojar el contenido que deseamos compartir con ustedes. Sin embargo, estamos trabajando diligentemente para adquirir uno lo antes posible.

Entendemos la importancia de contar con un servidor confiable que garantice el acceso continuo a la información que proporcionamos. Por ello, estamos explorando diversas opciones y evaluando las mejores soluciones disponibles en el mercado.

Tan pronto como tengamos un servidor en funcionamiento, actualizaremos este artículo para incluir toda la información relevante sobre su adquisición y las mejoras que esto implicará en nuestra plataforma. Apache

Apache

Apache, es el servidor HTTP de código abierto más utilizado en el mundo. Tras una pequeña configuración, hemos logrado integrar Apache en nuestro entorno de servidor, garantizando así un rendimiento óptimo y una estabilidad excepcional para nuestros servicios en línea. Como es muy conocido y sencilla tanto la instalación como el funcionamiento no indagaremos mucho sobre su uso en este entorno, sin embargo mencionar que se ha gestionado de tal manera que se ha configurado para poder navegar entre directorios y servicios de manera mucho más sencilla.

Hosting

Tanto la página web como el resto de servicios están accesibles mediante el dominio espetoneta.es como se podrá apreciar. Este dominio ha sido comprado y mantenido mediante el proveedor OVH, el funcionamiento es un tanto complejo sin embargo mostraré un pequeño resumen de como se ha realizado toda la configuración de red.

Desde el dashboard de OVH podemos cambiar la dirección IP a la que apunte la zona DNS de tipo A, esta es la encargada de redirigir a nuestro router.

Routers y DMZ

Una vez tenemos configurada las zonas DNS de neustro dominio podemos pasar a la configuración del Router, a esta si estamos en la misma red del router podemos acceder mediante la IP 192.168.1.1 en el navegador, en este caso la configuración está hecha desde un router de la marca Orange.

Ya dentro de la configuración de nuestro router debemos cambiar la zona desmilitarizada o DMZ, añadiendo la IP de nuestro servidor, que en nuestro caso es una IP estática fija o estática que se ha establecido manualmente a la 192.168.1.10 para poder reconocerla fácilmente.

Una vez tenemos las zonas desmilitarizadas y las IPs configuradas correctamente simplemente debemos configurar el router para que redirija ciertas peticiones a la IP solicitada que en este caso es el protocolo TCP con los puertos 443 tanto interno como externo que son las que usa el servicio HTTPS, de esta manera todas las peticiones externas que reciba de este tipo las va a redirijir a la IP indicada que en este caso es la de nuestro servidor. Este paso es opcional, pues nuestro servidor ya será accesible mediante la zona desmilitarizada sin embargo de esta manera podemos asegurarnos de que si se realiza otra petición esta será dirigida directamente a nuestro servidor.

Raspberry Pi Zero / 4

Raspberry Pi 4

Para este proyecto hemos usado una Raspberry Pi 4 Modelo B que dispone de 8GB de memoria RAM, esta será la Raspberry administradora del resto de Raspberrys que estarán en nuestro servidor.

Esta Raspberry Pi 4 la montaremos en una carcasa modelada en 3D por nosotros diseñada para poder ser montada en racks junto a servidores más complejos, este diseño se ha realizado en Blender y se ha impreso en 3D en una Ender 3 Pro V2. Las configuraciones de impresión son las siguientes

    Detalles de impresión
  • Cabezal: 4mm
  • Calidad: Ultra
  • Material: PLA+
  • Detalles: Modelo "sliced" en Ultimaker Cura.

El modelo 3D final puede variar con el resultado final.

Dentro de la Raspberry Pi 4 montaremos un Ubuntu Live Server, para no complicar demasiado los entornos de instalación de SO ni complicaciones con el resto de API y conexiones con el resto de placas y servidores.

Como se ha mencionado con anterioridad, la Raspberry estará montada en el rack conectada 24/7 tanto al servidor principal como al resto de placas. Por lo que necesitamos una alta eficiencia térmica, ya que soportará el bot de Telegram y otros servicios por lo que en cierta medida la carga del procesador puede llegar a generar altas temperaturas y poder llegar a apagar el sistema o en la peor situación estropear el procesador por lo que hemos aplicado pads térmicos y disipadores pasivos, más el ventilador que está montado en la carcasa.



Raspberry Pi Zero

Para las Raspberry Pi Zero tenemos un proyecto menos ambicioso, pero también interesante, hemos pensado en introducir un sensor de temperatura DHT11 para que pueda monitorear la temperatura y humedad del ambiente del rack o sala donde estén todos los sistemas, de esta manera podremos tener un registro del ambiente.

Al no disponer de mucha potencia ni capacidad de procesamiento de datos, funcionamiento por WiFi entre otras limitaciones, a esta placa le pondremos un Ubuntu server base únicamente con samba y Python, de esta manera podremos mantener un registro de la temperatura del rack para obtenerlo todo controlado de esta manera podremos pasar archivos de temperatura por samba de manera individual y no restarle porcentaje de uso de procesador al resto de sistemas.

Cluster Final

Finalmente instalaremos todo en la caja impresa mencionada anteriormente, dentro de esta se ubicarán la Raspberry Pi 4, Raspberry Pi Zero, Disipadores y 3 ventiladores de 24mm con la capacidad de mantener la temperatura de ambas en standby entorno unos 30°, si en ambas hacemos un benchmark para aumentar la temperaura y el estrés del procesador hemos apreciado que la Raspberry Pi 4 máximo llega a 41° mientras que la Pi Zero no llega a pasar de los 31°, esto es debido a la potencia y capacidad de computo de ambas placas.

Bot Telegram

Este BOT está diseñado especificamente para poder obtener información, controlar y administrar el servidor de manera remota, el bot está creado usando Python3, a continuación mostraremos una pequeña demostración del código.

Inicio y API

Primeramente importamos la librería de telebot para poder conectarnos a la API de telegram mediante el token que se define más adelante, a continuación indicamos que de la carpeta comandos importemos los siguientes archivos los que contienen los comandos.

Finalmente definimos el token del bot que se nos haya especificado anteriormente en el botfather, este token lo definimos en una variable para poder usarlo repetidas veces más adelante sin necesidad de repetir el token completo a lo largo del código.

Comandos y funciones

El bot, como cualquier otro funciona por comandos, estos hemos decidido separarlos por archivos individuales para poder obtener una mucho mejor organización y mejor distribución de contenido. Los comandos tinen un funcionamiento simple pero complejo a su vez, por parte del archivo principal, el que mantiene "vivo" al bot, es decir main.py, llamará mediante funciones a los archivos que se han especificado en la tercera línea de código del archivo main.py

Como se ha mencionado anteriormente está el archivo principal "main.py" que llama a otros archivos individuales que contienen el código de los comandos, por ende es importante tener bien definidas las rutas de los comandos.

Ahora mostraremos un comando de ejemplo como puede ser el comando /ping, este comando como su nombre indica lo que realiza es mandar un paquete de consulta ICMP a la ip o dirección indicada, esto en este caso no tiene relevancia, sin embargo deberemos tener en cuenta que el código en el interior del comando será ejecutado por separado y el resultado devuelto al archivo principal por lo que siempre que se hagan comandos demasiado complejos puede que tengan resoluciones erroneas.
